# الدليل الفني - Technical Documentation

## معمارية النظام (System Architecture)

### تدفق الطلب (Request Flow)

```
طلب المستخدم
     ↓
index.php (Root) - يخدم الملفات الثابتة من public/
     ↓ (إذا لم يكن ملف ثابت)
public/index.php - يبدأ الجلسة ويحمل الإعدادات
     ↓
App\Config\Router - يوزع الطلب على الـ Controller المناسب
     ↓
Controller - يعالج الطلب، يتفاعل مع Models
     ↓
Model - يتفاعل مع قاعدة البيانات عبر PDO
     ↓
View - يعرض الصفحة باستخدام Layout system
```

### نظام الـ Routing

الملف: `app/Config/Router.php`

- يتم تعريف المسارات في `config/routes.php` كمصفوفة
- تحويل الأنماط مثل `{id}` إلى تعبيرات منتظمة Regex
- استخراج الباراميترات وإضافتها إلى `$_GET` و `$_REQUEST`
- دعم Middleware chain قبل تنفيذ الـ Controller
- دعم تحديد HTTP method (GET, POST)

مثال على تعريف مسار:
```php
'/courses/{id}' => [
    'controller' => 'CourseController',
    'action' => 'show',
    'method' => 'GET',
    'middleware' => ['Auth'] // اختياري
]
```

### نظام قاعدة البيانات

الملف: `app/Config/Database.php`

- اتصال PDO (نمط Singleton)
- وضع الجلب: `PDO::FETCH_OBJ`
- وضع الأخطاء: استثناءات (Exceptions)
- دوال مساعدة: `raw()`, `fetch()`, `fetchAll()`, `insert()`, `update()`, `delete()`

```php
// أمثلة على استخدام Database
$users = Database::fetchAll("SELECT * FROM users WHERE role = ?", ['admin']);
$user = Database::fetch("SELECT * FROM users WHERE id = ?", [1]);
Database::insert("INSERT INTO users (name, email) VALUES (?, ?)", ['Ali', 'ali@test.com']);
```

### نظام الـ Model

الملف: `app/Core/Model.php`

كل Model يمدد الـ Base Model ويحدد `$table` و `$primaryKey`:

```php
class User extends Model {
    protected static $table = 'users';
    protected static $primaryKey = 'id';
}
```

الدوال المتاحة (static):
- `all()` - جميع السجلات
- `find($id)` - سجل واحد
- `where($conditions)` - استعلام بشرط (يعيد Query Builder)
- `whereFirst($column, $value)` - أول سجل مطابق
- `create($data)` - إنشاء سجل
- `updateRecord($id, $data)` - تحديث سجل
- `deleteRecord($id)` - حذف سجل
- `count()` - عدد السجلات
- `paginate($page, $perPage)` - تصفح الصفحات

كائن Query Builder يدعم:
- `get()` - تنفيذ وجلب النتائج
- `first()` - أول نتيجة فقط
- `count()` - عدد النتائج
- `orderBy($column, $dir)` - ترتيب

### نظام العرض (View System)

الملف: `app/Core/View.php`

```php
View::render('front/home', $data);
View::render('admin/dashboard', $data);
View::render('auth/login'); // بدون layout
```

اختيار الـ Layout يتم تلقائياً حسب المسار:
- `admin/*` → يستخدم `layouts/admin.php`
- `auth/*` → بدون layout
- الباقي → يستخدم `layouts/front.php`

إذا كان الطلب Ajax (`X-Requested-With: XMLHttpRequest`) يُعاد المحتوى فقط بدون Layout.

### نظام الـ Middleware

الملف: `app/Core/Middleware.php`

```php
class Auth extends Middleware {
    public function handle(): bool {
        if (!isset($_SESSION['user_id'])) {
            redirect('/login');
            return false;
        }
        return true;
    }
}
```

يتم تسجيل الـ Middleware في `public/index.php`:
```php
$router->registerMiddleware('Auth', \App\Middleware\Auth::class);
$router->registerMiddleware('Admin', \App\Middleware\Admin::class);
```

### نظام التحقق (Validation)

الملف: `app/Core/Controller.php`

```php
$this->validate($data, [
    'name' => 'required|min:3|max:100',
    'email' => 'required|email',
    'age' => 'numeric',
]);
```

القواعد المدعومة: `required`, `min:N`, `max:N`, `email`, `numeric`

### مساعدات عامة (Helpers)

الملف: `app/Config/Helpers.php`

| الدالة | الوصف |
|--------|-------|
| `env($key, $default)` | قراءة متغير بيئة | 
| `asset($path)` | رابط لملف ثابت |
| `url($path)` | رابط كامل للتطبيق |
| `redirect($url)` | إعادة توجيه |
| `csrf_token()` | توليد token CSRF |
| `csrf_field()` | hidden input لـ CSRF |
| `verify_csrf()` | التحقق من CSRF |
| `flash($key, $value)` | رسالة وميضية |
| `format_date($date)` | تنسيق تاريخ |
| `log_activity(...)` | تسجيل نشاط |

## الصور والملفات الثابتة

### خدمة الملفات الثابتة

الملف `index.php` في الجذر يخدم الملفات من `public/` مباشرة:
```php
$mimeTypes = [
    'css' => 'text/css', 'js' => 'application/javascript',
    'png' => 'image/png', 'jpg' => 'image/jpeg',
    // ...
];
```

### رفع الملفات

Trait: `app/Traits/Uploadable.php`

```php
use App\Traits\Uploadable;

class FileController extends Controller {
    use Uploadable;
    
    public function store() {
        $path = $this->uploadFile($_FILES['file'], 'files');
        // $path = 'public/uploads/files/file_<uniqid>.pdf'
    }
}
```

طريقة التخزين:
- `public/uploads/files/` - ملفات تعليمية
- `public/uploads/settings/` - صور الإعدادات

الأمان:
- `public/.htaccess` يمنع الوصول المباشر لـ `assets/uploads/`
- الملفات تُخدم عبر `FileController::download()` و `FileController::preview()`

## دليل النشر (Deployment Guide)

### التثبيت على 000WebHost

1. أنشئ حساب على https://www.000webhost.com
2. ارفع جميع الملفات إلى `public_html/` (باستثناء `.env.production`)
3. أنشئ قاعدة بيانات MySQL من لوحة التحكم
4. استورد `database/schema.sql` عبر phpMyAdmin
5. عدل `.env`:
   ```
   APP_ENV=production
   APP_URL=https://your-site.000webhostapp.com
   APP_DEBUG=false
   DB_HOST=localhost
   DB_NAME=your_db_name
   DB_USER=your_db_user
   DB_PASS=your_db_password
   ```
6. احذف ملف `.env.production` من السيرفر

### التثبيت على InfinityFree (بديل أفضل)

نفس خطوات 000WebHost لكن:
- لا توجد إعلانات
- أداء أفضل واستقرار أعلى
- مساحة 5GB + قاعدة بيانات MySQL

## الإضافات والتطوير

### إضافة Controller جديد

```php
// 1. أنشئ الملف app/Controllers/NewController.php
namespace App\Controllers;
use App\Core\Controller;

class NewController extends Controller {
    public function index() {
        $this->view('front/new', ['title' => 'صفحة جديدة']);
    }
}

// 2. أضف المسار في config/routes.php
'/new-page' => [
    'controller' => 'NewController',
    'action' => 'index',
    'method' => 'GET',
]
```

### إضافة Middleware جديد

```php
// 1. أنشئ الملف app/Middleware/NewMiddleware.php
namespace App\Middleware;
use App\Core\Middleware;

class NewMiddleware extends Middleware {
    public function handle(): bool {
        // منطق التحقق
        return true;
    }
}

// 2. سجله في public/index.php
$router->registerMiddleware('New', \App\Middleware\NewMiddleware::class);

// 3. استخدمه في config/routes.php
'/protected' => [
    'controller' => 'SomeController',
    'action' => 'index',
    'method' => 'GET',
    'middleware' => ['Auth', 'New']
]
```
