## الفصل الأول: مقدمة عن المشروع

### 1.1 نظرة عامة

نظام اللجنة العلمية هو تطبيق ويب متكامل لإدارة المحتوى التعليمي الخاص باللجان العلمية في الجامعات. يهدف النظام إلى تنظيم المواد الدراسية والملفات التعليمية بطريقة سهلة ومنظمة، مع توفير واجهة تفاعلية للطلاب وأعضاء هيئة التدريس.

### 1.2 أهداف النظام

1. **تنظيم المحتوى التعليمي**: تصنيف المواد الدراسية حسب التخصص والمستوى والفصل
2. **إدارة الملفات التعليمية**: رفع وتصنيف وتحميل المحاضرات والملخصات ونماذج الاختبارات
3. **تقييم المواد**: تمكين الطلاب من تقييم المواد الدراسية
4. **إدارة الإعلانات**: نشر الإعلانات الهامة مع إمكانية الجدولة والتثبيت
5. **التحكم بالصلاحيات**: نظام أدوار متكامل (Admin, Manager, Guest)
6. **سهولة البحث**: إمكانية البحث عن المواد الدراسية بفلترة متعددة

### 1.3 التقنيات المستخدمة

| التقنية | الإصدار | الاستخدام |
|---------|---------|-----------|
| PHP | >= 7.4 | لغة البرمجة الأساسية |
| MySQL | >= 5.7 | قاعدة البيانات |
| Apache | - | خادم الويب مع mod_rewrite |
| Bootstrap 5.3.2 | RTL | إطار الواجهة الأمامية (دعم العربية) |
| Font Awesome 6.5.1 | - | أيقونات |
| Chart.js 4.4.1 | - | رسوم بيانية في لوحة التحكم |
| Composer | - | إدارة اعتماديات PHP |

### 1.4 المكتبات الخارجية (Composer)

| المكتبة | الوظيفة |
|---------|---------|
| vlucas/phpdotenv | إدارة متغيرات البيئة عبر ملف .env |
| phpmailer/phpmailer | إرسال البريد الإلكتروني (جاهز للاستخدام) |
| symfony/polyfill-php80 | توافق مع PHP 8.0 |

### 1.5 فريق العمل

- **نوع المشروع**: نظام ويب مفتوح المصدر
- **الجهة المستفيدة**: اللجنة العلمية بالجامعة
- **لغة الواجهة**: العربية (RTL) مع دعم الوضع الليلي

## الفصل الثالث: شرح التطبيق والواجهات

### 3.1 آلية عمل النظام

عند دخول المستخدم إلى الموقع، تتم معالجة الطلب كالتالي:

```
1. المتصفح يرسل طلب (مثال: /courses/5)
2. index.php (الجذر) يتأكد أن الملف ليس ملف ثابت
3. public/index.php يبدأ الجلسة ويحمل الإعدادات
4. Router يبحث عن مسار مطابق في config/routes.php
5. إذا وجد التطابق:
   a. ينفذ Middleware إن وجد
   b. يستدعي الـ Controller المناسب
   c. Controller يستخدم Models للتفاعل مع قاعدة البيانات
   d. Controller يمرر البيانات إلى View
   e. View يعرض الصفحة مع الـ Layout المناسب
6. إذا لم يجد مسار: يعرض صفحة 404
```

### 3.2 الواجهة الأمامية (الموقع العام)

#### 1. الصفحة الرئيسية `/`
```
المحتوى:
├── شريط ترحيبي مع إحصائيات
│   ├── عدد التخصصات
│   ├── عدد المواد
│   ├── عدد الملفات
│   └── عدد الإعلانات
├── شبكة التخصصات (بطاقات)
│   └── كل بطاقة تحتوي: اسم التخصص، وصف، عدد المواد
├── الإعلانات المثبتة
└── عند النقر على تخصص → نافذة اختيار المستوى
```

#### 2. تفاصيل التخصص `/majors/{id}`
```
المحتوى:
├── اسم التخصص ووصفه
├── أزرار فلترة المستويات
└── المواد مقسمة حسب المستوى والفصل
    └── كل مادة: اسم + رابط للتفاصيل
```

#### 3. تفاصيل المادة `/courses/{id}`
```
المحتوى:
├── معلومات المادة (تخصص، مستوى، فصل)
├── نظام التقييم (5 نجوم)
│   ├── عرض متوسط التقييم
│   ├── رسم بياني لتوزيع التقييمات
│   └── إمكانية التقييم (مرة واحدة لكل زائر)
├── الملفات التعليمية
│   ├── تبويبات حسب النوع (محاضرات، ملخصات، نماذج، اختبارات)
│   ├── ترتيب: الأحدث أو الأكثر تحميلاً
│   └── كل ملف: اسم، حجم، تاريخ، زر تحميل ومعاينة
└── تقييمات المستخدمين
```

#### 4. البحث `/search`
```
المحتوى:
├── حقل بحث مع فلترة
│   ├── كلمة مفتاحية
│   ├── فلترة حسب التخصص
│   ├── فلترة حسب المستوى
│   └── فلترة حسب الفصل
└── نتائج البحث (بطاقات المواد)
```

#### 5. الإعلانات `/announcements`
```
المحتوى:
├── الإعلانات المثبتة (في الأعلى)
└── باقي الإعلانات (حسب التاريخ)
    └── كل إعلان: عنوان، تاريخ، محتوى، صورة (اختياري)
```

| **Admin** | وصول كامل للنظام | جميع صفحات الإدارة |
| **Manager** | صلاحيات محدودة حسب تخصصه/مستواه | Dashboard، المواد، الملفات، الإعلانات (إضافة فقط)، المشرفون |
| **Guest** | لا يمكنه الدخول للوحة التحكم | الموقع العام فقط |

#### صلاحيات Manager بالتفصيل:

```
المشرف (Manager) يستطيع:
├── مشاهدة لوحة الإحصائيات
├── إدارة المواد (ضمن نطاقه فقط)
├── إدارة الملفات (ضمن نطاقه فقط)
├── إضافة إعلانات جديدة
└── إدارة المشرفين التابعين له

المشرف (Manager) لا يستطيع:
├── إدارة التخصصات
├── إدارة المستويات
├── إدارة الفصول
├── إدارة المستخدمين
├── تعديل/حذف إعلانات غيره
├── تعديل إعدادات الموقع
└── حذف إعلانات
```

### 3.5 نظام تقييم المواد

```
آلية التقييم:
1. المستخدم يفتح صفحة المادة
2. يظهر متوسط التقييم الحالي مع عدد المصوتين
3. المستخدم يضغط على نجمة (1-5)
4. يتم إرسال طلب AJAX إلى /courses/{id}/rate
5. النظام يتحقق:
   a. المادة موجودة
   b. التقييم بين 1 و 5
   c. المستخدم لم يقيم من قبل (حسب IP + Session)
6. إذا نجح التحقق: يسجل التقييم ويعيد المتوسط الجديد
7. إذا فشل: يعيد رسالة خطأ

مكافحة التكرار:
├── UNIQUE constraint في قاعدة البيانات (course_id, ip_address, session_id)
├── localStorage في المتصفح (يمنع إعادة التقييم)
└── التحقق من IP + Session
```

### 3.6 نظام رفع الملفات

```
آلية الرفع:
1. المستخدم يملأ نموذج رفع ملف (عنوان، نوع، وصف، ملف)
2. النظام يتحقق:
   a. CSRF token صحيح
   b. جميع الحقول المطلوبة موجودة
   c. الملف رفع بنجاح
   d. الامتداد مسموح
   e. الحجم لا يتجاوز الحد المسموح
3. يتم إنشاء اسم فريد: file_<uniqid>.<ext>
4. نقل الملف إلى public/uploads/files/
5. حفظ البيانات في قاعدة البيانات
6. تسجيل النشاط في activity_logs

الأمان:
├── الوصول المباشر للمجلد محظور عبر .htaccess
├── الملفات تُخدم فقط عبر FileController
└── التحقق من صلاحية المستخدم
```

### 3.7 نظام الإعلانات

```
آلية الإعلانات:
1. الإعلانات النشطة فقط تظهر في الموقع العام
2. دعم الجدولة (تاريخ بداية ونهاية)
3. الإعلانات المثبتة تظهر في الصفحة الرئيسية
4. ترتيب العرض: المثبت أولاً ثم الأحدث
5. أي مستخدم مسجل يمكنه إضافة إعلان
6. فقط Admin يمكنه تعديل أو حذف إعلانات الآخرين
```

### 3.8 نظام تسجيل النشاطات (Activity Log)

```
يتم تسجيل كل عملية مهمة في النظام:
├── تسجيل الدخول / تسجيل الخروج
├── إنشاء سجل جديد (create)
├── تحديث سجل (update)
├── حذف سجل (delete)
└── كل سجل يحتوي: مستخدم، نوع العملية، الجدول، السجل، تفاصيل، IP، المتصفح
```

### الملحق 1: هيكل قاعدة البيانات الكامل

```sql
-- التخصصات
CREATE TABLE majors (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL UNIQUE,
    description TEXT,
    icon VARCHAR(50) DEFAULT 'fa-graduation-cap',
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- المستويات
CREATE TABLE levels (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL UNIQUE,
    level_number INT NOT NULL UNIQUE,
    sort_order INT DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- الفصول
CREATE TABLE semesters (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL UNIQUE,
    semester_number INT NOT NULL UNIQUE,
    sort_order INT DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- المواد الدراسية
CREATE TABLE courses (
    id INT PRIMARY KEY AUTO_INCREMENT,
    major_id INT NOT NULL,
    level_id INT NOT NULL,
    semester_id INT NOT NULL,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (major_id) REFERENCES majors(id) ON DELETE CASCADE,
    FOREIGN KEY (level_id) REFERENCES levels(id) ON DELETE CASCADE,
    FOREIGN KEY (semester_id) REFERENCES semesters(id) ON DELETE CASCADE
);

-- المستخدمون
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    major_id INT NULL,
    full_name VARCHAR(150) NOT NULL,
    email VARCHAR(150) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    phone VARCHAR(20),
    managed_level_id INT NULL,
    managed_major_id INT NULL,
    role ENUM('admin', 'manager', 'guest') DEFAULT 'guest',
    is_active BOOLEAN DEFAULT TRUE,
    last_login TIMESTAMP NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (major_id) REFERENCES majors(id) ON DELETE SET NULL,
    FOREIGN KEY (managed_level_id) REFERENCES levels(id) ON DELETE SET NULL,
    FOREIGN KEY (managed_major_id) REFERENCES majors(id) ON DELETE SET NULL
);

-- الملفات التعليمية
CREATE TABLE files (
    id INT PRIMARY KEY AUTO_INCREMENT,
    course_id INT NOT NULL,
    uploaded_by INT NOT NULL,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    file_name VARCHAR(255) NOT NULL,
    file_path VARCHAR(500) NOT NULL,
    file_size BIGINT DEFAULT 0,
    file_extension VARCHAR(10),
    file_type ENUM('lecture', 'summary', 'model', 'exam', 'other') DEFAULT 'other',
    download_count INT DEFAULT 0,
    is_approved BOOLEAN DEFAULT TRUE,
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (course_id) REFERENCES courses(id) ON DELETE CASCADE,
    FOREIGN KEY (uploaded_by) REFERENCES users(id) ON DELETE CASCADE
);

-- الإعلانات
CREATE TABLE announcements (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    image_path VARCHAR(500) NULL,
    is_active BOOLEAN DEFAULT TRUE,
    is_pinned BOOLEAN DEFAULT FALSE,
    starts_at TIMESTAMP NULL,
    expires_at TIMESTAMP NULL,
    created_by INT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE CASCADE
);

-- التقييمات
CREATE TABLE ratings (
    id INT PRIMARY KEY AUTO_INCREMENT,
    course_id INT NOT NULL,
    rating TINYINT NOT NULL CHECK (rating >= 1 AND rating <= 5),
    ip_address VARCHAR(45),
    session_id VARCHAR(128),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (course_id) REFERENCES courses(id) ON DELETE CASCADE,
    UNIQUE KEY unique_vote (course_id, ip_address, session_id)

-- الإعدادات
CREATE TABLE settings (
    id INT PRIMARY KEY AUTO_INCREMENT,
    setting_key VARCHAR(100) UNIQUE NOT NULL,
    setting_value TEXT,
    setting_type ENUM('text', 'textarea', 'image', 'color', 'url') DEFAULT 'text',
    setting_group VARCHAR(50) DEFAULT 'general',
    is_public BOOLEAN DEFAULT FALSE,
```
